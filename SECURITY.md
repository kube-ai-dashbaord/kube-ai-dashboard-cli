# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within **k13s**, please follow these steps:

1. **Do not** open a public GitHub issue.
2. Send a detailed report to the maintainers at [your-email@example.com].
3. Include information on how to reproduce the vulnerability.
4. We will acknowledge your report and work on a fix.

We follow a responsible disclosure policy and request that you give us time to address the issue before making it public.
