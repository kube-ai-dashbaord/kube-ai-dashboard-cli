Getting Started

1

Installation

# Build from source
git clone https://github.com/kube-ai-dashbaord/kube-ai-dashboard-cli.git
cd kube-ai-dashboard-cli
go build -o k13s ./cmd/kube-ai-dashboard-cli/main.go

# Or download binary (coming soon)
# curl -LO https://github.com/.../releases/latest/download/k13s
# chmod +x k13s && sudo mv k13s /usr/local/bin/
2

Configure AI (Optional)

# Create config file
mkdir -p ~/.config/k13s
cat > ~/.config/k13s/config.yaml << EOF
llm:
  provider: openai
  model: gpt-4
  api_key: your-api-key
  # Or use Ollama for local AI:
  # provider: ollama
  # endpoint: http://localhost:11434/v1
  # model: llama2

language: en
EOF
3

Run k13s

# TUI Mode (default)
./k13s

# Web UI Mode
./k13s -web -port 8080
# Then open http://localhost:8080

Resource Navigation (Enter Key)

When you press Enter on a resource, k13s navigates to related resources:

FromNavigates ToFilter Applied

CronJobJobsJobs created by CronJob
NodePodsPods running on node (all namespaces)

Press Esc to go back to the previous view. Navigation history is maintained.

Configuration

Configuration is stored in ~/.config/k13s/config.yaml

Full Configuration Example

# LLM Configuration
llm:
  provider: openai          # openai, ollama, anthropic
  model: gpt-4              # Model name
  endpoint: ""              # Custom endpoint (optional)
  api_key: "sk-..."         # API key (or use env: OPENAI_API_KEY)
  temperature: 0.7          # Response creativity (0-1)
  max_tokens: 2048          # Max response length

# UI Settings
language: en                # en, ko, zh, ja
beginner_mode: false        # Simpler AI explanations
theme: dark                 # dark, light

# Features
enable_audit: true          # Log all actions to SQLite
log_level: info             # debug, info, warn, error

# Web UI Authentication (optional)
auth:
  enabled: true
  session_timeout: 24h
  users:
    - username: admin
      password_hash: "..."   # bcrypt hash
      role: admin

# LDAP Integration (optional)
  ldap:
    enabled: false
    host: ldap.example.com
    port: 389
    use_tls: true
    bind_dn: cn=service,dc=example,dc=com
    bind_password: secret
    base_dn: dc=example,dc=com
    user_search_filter: "(uid=%s)"
    admin_groups:
      - k8s-admins
    user_groups:
      - k8s-users
