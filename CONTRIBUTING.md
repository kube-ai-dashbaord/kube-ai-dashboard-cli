# Contributing to k13s

We love your input! We want to make contributing to **k13s** as easy and transparent as possible.

## Our Development Process
- All development happens on GitHub.
- We use GitHub Issues to track bugs and features.
- We follow the standard GitHub Flow for contributions.

## Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`go test ./...`).
5. Make sure your code lints.

## Coding Style
- Follow standard Go conventions (`gofmt`).
- Use descriptive variable and function names.
- Document public APIs with comments.
